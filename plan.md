# 《庄子》网站设计与开发计划 plan.md

## 1. 项目概述

### 1.1 项目名称

**庄子宇宙**

### 1.2 副标题

**在逍遥、齐物与无用之间，重新理解自由**

### 1.3 网站定位

本项目不是一个普通的《庄子》在线阅读网站，而是一个以“故事进入哲学”的沉浸式古籍学习网站。

《庄子》的核心魅力不在于条文式说教，而在于寓言、人物、想象、悖论、辩论和精神自由。因此网站的设计重点应当是：

> 先让用户被故事吸引，再逐步理解背后的哲学思想。

网站应同时具备以下能力：

1. 原文阅读
2. 白话翻译
3. 注释解释
4. 寓言故事
5. 思想地图
6. 人物图谱
7. 现代生活解释
8. 每日一则庄子
9. 搜索与索引
10. 沉浸式东方哲学视觉体验

---

## 2. 项目目标

### 2.1 核心目标

把《庄子》做成一个适合现代人阅读、理解、记忆和使用的网站。

用户进入网站后，不只是“查原文”，而是能够完成以下路径：

```text
被首页气质吸引
→ 通过故事产生兴趣
→ 阅读原文与白话
→ 理解思想概念
→ 看到人物关系
→ 联系现代生活
→ 形成自己的理解
```

### 2.2 用户目标

网站主要服务三类用户：

#### 1. 初学者

这类用户可能没有古文基础，只知道“庄周梦蝶”“庖丁解牛”等故事。

他们需要：

- 看得懂
- 读得下去
- 先看故事，再看原文
- 用现代语言解释庄子思想

#### 2. 系统学习者

这类用户希望完整理解《庄子》的结构和思想。

他们需要：

- 内篇、外篇、杂篇完整结构
- 思想脉络
- 概念关联
- 人物关系
- 章节之间的联系

#### 3. 对设计和东方哲学感兴趣的用户

这类用户希望网站本身具有审美和体验感。

他们需要：

- 空灵、留白、东方气质
- 沉浸式动效
- 现代网页交互
- 不是传统古籍网站的枯燥排版

## 3. 设计总原则

### 3.1 从故事进入哲学

《庄子》应以寓言故事作为主要入口。

用户不一定一开始就能理解“逍遥”“齐物”“无为”“道”，但通常能记住：

- 鲲鹏展翅
- 庄周梦蝶
- 庖丁解牛
- 濠梁之辩
- 无用之树
- 井底之蛙
- 螳臂当车
- 鼓盆而歌

所以网站首页、导航和内容结构都要突出“故事入口”。

### 3.2 原文不孤立出现

每一段原文都应该配套：

- 白话翻译
- 关键词
- 注释
- 思想解释
- 相关故事
- 现代理解

不要让用户面对一整屏古文。

### 3.3 强调“精神自由”

《庄子》网站的精神核心不是“成功学”，不是“管理学”，也不是“鸡汤”，而是：

> 你不一定非要活在别人规定的尺度里。

网站所有设计都应围绕：

- 逍遥
- 自由
- 反功利
- 反成见
- 顺应自然
- 无用之用
- 生死一体
- 万物齐一

### 3.4 视觉要空，不要满

庄子的气质是开阔、流动、虚静。

因此页面设计不应密集、厚重、复杂，而应：

- 大面积留白
- 低饱和颜色
- 慢节奏动效
- 墨色与米白为主
- 山、水、云、风、蝶、鱼、鹏等意象

## 6. 阅读模块设计

### 6.1 阅读模块目标

让用户系统阅读《庄子》，但不被古文劝退。

### 6.2 章节结构

传统上《庄子》分为：

```text
内篇
外篇
杂篇
```

内篇包括：

```text
1. 逍遥游
2. 齐物论
3. 养生主
4. 人间世
5. 德充符
6. 大宗师
7. 应帝王
```

外篇和杂篇可按照通行篇目继续扩展。

### 6.3 阅读入口页 `/read`

页面包含：

- 推荐阅读路径
- 内篇、外篇、杂篇入口
- 新手推荐
- 按主题阅读
- 按故事阅读

新手推荐路径：

```text
1. 逍遥游：理解精神自由
2. 齐物论：理解是非相对
3. 养生主：理解顺势而行
4. 人间世：理解乱世自处
5. 大宗师：理解生死与自然
```

### 6.4 章节详情页 `/chapter/[slug]`

章节详情页建议使用三栏结构。

#### 左侧：章节目录

内容：

- 当前篇名
- 当前小节
- 段落列表
- 快速跳转
- 阅读进度

#### 中间：正文区域

内容：

- 原文
- 白话翻译
- 分段阅读
- 点击展开解释

#### 右侧：辅助理解区

内容：

- 本章关键词
- 相关寓言
- 相关人物
- 相关思想
- 现代启发

### 6.5 原文展示方式

不要整篇一次性展示，应按段落拆开。

每段结构：

```text
原文：
北冥有鱼，其名为鲲。

白话：
北方的大海里有一条鱼，名字叫鲲。

注释：
北冥：北方的大海。

思想解释：
这里不是在讲真实的大鱼，而是在打开一个超越日常尺度的想象空间。
庄子用鲲鹏表达“大知”和“小知”的差异。
```

### 6.6 段落交互

每段原文支持：

- 展开白话
- 展开注释
- 展开思想解释
- 查看相关故事
- 收藏该段
- 复制原文
- 切换繁简
- 切换阅读模式

### 6.7 阅读模式

提供三种阅读模式：

#### 1. 原文模式

只显示古文，适合熟悉文本的用户。

#### 2. 对照模式

原文 + 白话 + 注释，适合学习者。

#### 3. 沉浸模式

隐藏侧栏，只显示正文，背景更安静，适合长时间阅读。

```text
真正的高手，不是靠蛮力，而是顺着事物的纹理行动。

哲学含义：

当一个人真正理解事物本身的结构和规律时，他的行动就不再是硬碰硬，而是顺势而为。

现代启发：
写代码、做产品、处理人际关系也是如此。
初学者靠用力，高手靠理解结构。
```

### 10.1 模块目标

### 10.2 现代问题与庄子思想对应表

|---|---|---|
| 工作内卷 | 无用之用、逍遥 | 无用之树 |
| 被别人评价困住 | 齐物论、逍遥 | 庄周梦蝶 |
| 感到自己没价值 | 无用之用 | 支离疏 |
| 焦虑未来 | 安时处顺 | 大宗师 |

### 10.3 页面结构 `/modern`

- 现代问题列表

### 10.4 现代主题详情页示例

#### 主题：为什么你总是被评价困住？

```text
现代问题：
你很在意别人怎么看你，总是担心自己不够成功、不够优秀、不够有价值。

庄子可能会说：
你之所以痛苦，是因为你相信世界只有一种评价标准。

对应思想：
齐物、逍遥、无用之用

相关故事：
庄周梦蝶、无用之树

解释：
在《齐物论》里，庄子试图打破固定的是非标准。
很多时候，所谓好坏、成败、贵贱，并不是绝对真理，而是站在某个角度形成的判断。
当你只接受一种标准时，你就会被这种标准困住。
```

### 10.5 庄子式提问功能

设计一个互动问答入口。

用户选择当前困惑：

```text
我最近很焦虑

我被工作压得喘不过气
我很在意别人看法
我不知道人生要追求什么
我害怕失败
我陷入人际冲突
我对死亡感到恐惧
```

系统返回：

```text

推荐故事
推荐章节
一句庄子式回应
进一步阅读路径
```

示例：

```text
用户选择：我觉得自己没价值
```

## 17. 技术方案

### 17.1 推荐技术栈

```text
框架：Next.js
语言：TypeScript
样式：Tailwind CSS
内容：MDX + JSON
动效：Framer Motion
搜索：Fuse.js 或 MiniSearch
图谱：React Flow 或 D3.js
部署：Vercel / Netlify / 自有服务器
```

### 17.2 为什么选择 Next.js

原因：

- 适合内容型网站
- SEO 友好
- 支持静态生成
- 路由清晰
- 可以结合 MDX
- 后期可扩展 API
- 部署方便

### 17.3 为什么使用 MDX

原因：

- 古籍内容适合用 Markdown 管理
- 可在文章里插入 React 组件
- 方便后续编辑
- 适合静态生成
- 内容与代码相对分离

### 17.4 搜索方案

第一阶段使用前端本地搜索：

```text
Fuse.js
```

搜索数据构建为：

```text
search-index.json
```

后期如果内容变多，可升级为：

```text
Algolia
Meilisearch
Typesense
```

### 17.5 图谱方案

思想地图和人物关系图可以使用：

```text
React Flow
```

优点：

- 交互方便
- 节点可点击
- 适合概念图谱
- 开发成本比 D3 更低

## 22. 内容编辑规范

### 22.1 原文规范

原文要：

- 标明篇章
- 分段清晰
- 保持古文原貌
- 不随意改字
- 如有版本差异，后续可做版本说明

### 22.2 白话规范

白话翻译要：

- 准确
- 简洁
- 不过度发挥
- 尽量用现代中文表达

### 22.3 思想解释规范

思想解释要：

- 不鸡汤化
- 不成功学化
- 不强行现代化
- 但要能联系现实
- 明确说明这是一种解释，不是唯一答案

### 22.4 现代启发规范

现代启发要避免：

```text
庄子教你赚钱
庄子教你成功
庄子教你管理团队
庄子教你躺平
```

更适合表达为：

```text
庄子帮助你重新理解价值标准
庄子提醒你不要被单一尺度困住
庄子提供一种观察人生的方式
```

## 29. 最终定义

这个网站不应该只是：

```text
《庄子》在线阅读
```

而应该是：

```text
一个让现代人重新理解自由、价值、生死、功利和自我的哲学空间。
```

《论语》适合做成“人生修养系统”。

《道德经》适合做成“极简智慧系统”。

《易经》适合做成“变化推演系统”。

而《庄子》最适合做成：

```text
精神自由与想象力的网站
```

它的灵魂不是教人如何成功，而是提醒人：

```text
你不一定非要活在别人规定的尺度里。
```
